# AWS Transit Gateway

AWS Transit Gateway conecta múltiples VPCs y redes on-premises de forma centralizada.

---

## Qué problema resuelve
Evita arquitecturas complejas de VPC Peering.

---

## Características clave
- Hub-and-spoke.
- Escalable.
- Centraliza conectividad.

---

## Casos de uso
- Múltiples VPCs.
- Arquitecturas grandes.
- Entornos corporativos.

---

## Pistas de examen
- TGW ≠ VPC Peering.
- Centraliza redes.
- Escala mejor.

---

## Resumen rápido
Transit Gateway = **hub de conectividad**
